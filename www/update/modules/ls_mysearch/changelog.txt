[v1.1.31]
  + Добавлен поиск по блогам (поиск по заголовками и описаниям блогов)

[v1.1.30]
  * Небольшая оптимизация запросов

[v1.1.29]
  + Добавлено некие элементы релевантности: если задается несколько слов, и
    есть тексты, где эти слова идут подряд, то эти тексты показываются вверху
    результатов поиска

[v1.1.28]
  + Если искомые слова встречаются рядом, то они попадут в один фрагмент
    сниппета
  * Изменено значение параметра MYSEARCH_SKIP_ALL_TAGS - если он задан, то
    поиск в тегах все равно идет, но сами теги в результатах поиска не 
    показываются

[v1.1.27]
  * minor bugfix

[v1.1.26]
  * Исправлен баг, считающий в результатах выдачи неопубликованные топики
  * Исправлен баг в поиске комментариев с параметром MYSEARCH_SKIP_ALL_TAGS

[v1.1.24]
  + Добавлена опция исключения тегов из поиска

[v1.1.22]
	+ Возможность логгирования поисковых запросов (только при работе с Админпанелью
	  версии 1.2.95 и выше)
  * Корректная разбивка результата по страницам

[v1.1.22]
  + Многие настройки вынесены в конфигурационный файл, в т.ч. шаблоны определения
	  слов (цифры и буквы алфовита с поддержкой по умолчанию всех языков Европы и 
		экс-СССР)
	+ Добавлена опция временного отключения перехвата запроса поиска без удаления
	  модуля
	+ Добавлено формирование сниппетов для вывода результатов поиска (стоит по
	  умолчанию), но при желании можно использовать и старые режимы вывода топиков:
		короткий с катом, полный текст
	* Улучшен алгоритм \"подсветки\" найденных результатов с вынесением параметров
	  подсветки в файл конфигурации
	+ Добавлены свои шаблоны для вывода результатов поиска единообразно, что
	  для топиков, что для комментариев
	* Исправлен баг, когда \"ломалась\" кодировка при передаче искомой фразы
	  непосредственно из адресной строки
	* Мелкий багфикс
